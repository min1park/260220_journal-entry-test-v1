# UI 설계: CF정산표 작성 웹앱

**작성일**: 2026-03-14
**기획서**: docs/specs/2026-03-14-cf-reconciliation-app.md
**설계자**: Claude

## 1. 개요

### 1.1 목적
CF정산표 작성의 전 과정(TB 업로드 → CoA 매핑 → 정산표 편집 → 검증 → Excel 출력)을
직관적인 웹 UI로 제공하여, Excel 수작업 대비 오류를 줄이고 생산성을 높인다.

### 1.2 사용자
- CPA / 회계감사인, 기업 결산 담당자
- 데스크톱 전용 (1280px+ 해상도)
- Excel/스프레드시트에 익숙한 사용자

### 1.3 사용자 목표
1. TB 데이터로부터 CF정산표 골격을 빠르게 생성
2. 스프레드시트와 동일한 사용감으로 조정분개 입력
3. 실시간 검증으로 오류를 즉시 발견
4. 완성된 CF정산표를 Excel로 내보내기

## 3. 전체 레이아웃

### 3.1 앱 셸 구조
```
┌─────────────────────────────────────────────────────────┐
│  TopBar: 로고 | 프로젝트명 | 단계 표시 | 저장 | 설정    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    Main Content                         │
│              (각 단계별 화면 전환)                        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  BottomBar: 검증 요약 뱃지 | 이전/다음 단계 버튼         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 단계 네비게이션 (Stepper)
```
 ① 프로젝트  →  ② TB 업로드  →  ③ CoA 매핑  →  ④ CF정산표  →  ⑤ 완료
   [완료]          [완료]         [진행중]        [대기]       [대기]
```
- 각 단계는 클릭으로 직접 이동 가능 (이전 단계가 완료된 경우)
- 현재 단계: Blue 원형 + 굵은 텍스트
- 완료 단계: Green 체크 아이콘
- 대기 단계: Gray 원형

#### 자동 매핑 규칙 (UI에서 제안)
| 계정명 키워드 | 자동 매핑 |
|-------------|----------|
| 현금, 예금, MMF | 현금 🔒 |
| 매출채권, 미수금, 선급 | 영업-자산 |
| 감가상각누계, 대손충당금 | 영업-조정 |
| 토지, 건물, 기계, 비품 | 투자-유형 |
| 특허, 소프트웨어, 개발비 | 투자-무형 |
| 매입채무, 미지급, 예수금 | 영업-부채 |
| 차입금 | 재무 |
| 리스부채, 사용권 | 비현금 |
| 자본금, 잉여금, 자기주식 | 자본 🔒 |

- 🔒 표시: 자동 잠금 (사용자가 변경 불가 또는 변경 시 경고)
- 자동 매핑 후 사용자가 수동으로 변경 가능
- 미매핑 계정이 있으면 "다음" 버튼에 경고 표시

### 5.2 주요 컴포넌트 명세

#### GridCell (핵심 컴포넌트)
```
Props:
  - value: number | null
  - onChange: (value: number) => void
  - isEditable: boolean
  - isSubtotal: boolean

  - idle: 값 표시 (천단위 콤마, 음수 빨강 괄호)
```

## 10. 구현 노트

### 성능 고려사항
- **가상화 필수**: 100열 × 120행 = 12,000 셀. DOM 전체 렌더링 불가
  - 행 가상화: `@tanstack/react-virtual` (visible rows만 렌더)
  - 열 가상화: 같은 라이브러리로 visible columns만 렌더
- **Frozen pane**: CSS `position: sticky` 활용 (좌측 + 상단 고정)
- **입력 디바운스**: 셀 값 변경 시 검증 재계산은 100ms 디바운스
- **Zustand**: 전체 정산표 상태를 단일 스토어로 관리, immer 미들웨어

### 숫자 입력 UX
- 입력 시: 순수 숫자만 (콤마 없이)
- 표시 시: 천단위 콤마 + 음수 괄호
- 음수 입력: `-` 또는 `(` 접두어
- 붙여넣기: 콤마/괄호 자동 제거 후 숫자 파싱

### Excel 출력 품질
- 실무에서 바로 쓸 수 있는 수준의 서식
- 열 너비, 행 높이, 테두리, 셀 색상 모두 설정
- 숫자 포맷: `#,##0;(#,##0);"-"` (양수/음수/영)
- 검증행 조건부 서식 포함
