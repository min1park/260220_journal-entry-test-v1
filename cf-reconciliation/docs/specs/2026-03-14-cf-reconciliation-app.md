# 기능명: CF정산표 작성 웹앱

**작성일**: 2026-03-14
**상태**: 초안
**기획자**: Claude

## 1. 개요

### 1.1 목적
CPA 감사/결산 실무에서 현금흐름표(CF) 작성을 위한 CF정산표를 체계적으로 생성하는 웹 애플리케이션.
현재 Excel 수작업으로 진행하는 CF정산표 작성 프로세스를 자동화하여 오류를 줄이고 효율성을 높인다.

### 1.2 대상 사용자
- CPA / 회계감사인
- 기업 결산 담당자
- 재무보고 실무자

### 1.3 기대 효과
- 시산표(TB) 업로드만으로 CF정산표 골격 자동 생성
- BS 증감 ↔ CF 항목 매핑 오류 최소화 (자동 검증)
- 별도/연결 CF정산표 일관성 있는 작성
- Excel 출력으로 기존 업무 프로세스와 호환

## 2. 핵심 도메인 개념

### 2.1 CF정산표란?
재무상태표(BS)의 당기초/당기말 잔액 증감을 현금흐름표(CF)의 각 항목에 매핑하는 작업지(Working Paper).

**원리**: BS의 모든 계정 증감은 반드시 CF의 어딘가에 반영되어야 한다.
```
Σ(각 BS계정의 증감) = 0  (차변/대변 합계 일치)
Σ(각 CF항목에 배분된 증감) = 해당 BS계정의 증감  (계정별 검증)
```

### 2.2 CF정산표 구조 (실무 기준)

#### 행(Row) 구조
```
[헤더 영역]
  R1: 검증행 (모든 컬럼 = 0이어야 함)
  R2: 당기말 BS 잔액
  R3: 계정 분류명 (재무제표 수준)
  R5: 컬럼별 검증 (재계산 차이 = 0)
  R6: 상세 계정과목명
  R7: 당기초 BS 잔액
  R8: 당기말 BS 잔액 (= R2)
  R9: 증감 (= R8 - R7)

[CF 본문]
  I. 영업활동 현금흐름
    1. 영업에서 창출된 현금흐름
      (1) 당기순이익
      (2) 조정항목 (감가상각비, 대손상각비, 이자비용, 외환차손익, ...)
      (3) 영업활동 자산부채 변동 (매출채권, 재고자산, 매입채무, ...)
    2. 이자수취 / 이자지급 / 배당금수취 / 법인세납부

  II. 투자활동 현금흐름
    (금융상품, 대여금, 유형자산, 무형자산, 투자부동산 취득/처분)

  III. 재무활동 현금흐름
    (차입금, 리스부채, 자기주식, 배당금지급)

  IV. 현금순증감 (= I + II + III)
  V. 기초 현금
  VI. 기말 현금
  환율변동효과

[비현금 거래]
  사용권자산/리스부채 신규, 유동성대체, 계정대체 등
```

#### 열(Column) 구조
```
A: 검증 (행별 차이 = 0)
B: 회사/매핑코드
C: 원장 계정명
D: CF 항목명
E: CF 금액 (= 해당 행의 F~끝 합계)

F ~ 끝: 개별 BS 계정 컬럼
  - 현금및현금성자산, 정기예금, 매출채권, 미수금, ...
  - 유형자산(토지, 건물, 기계, ...) + 감가상각누계액
  - 무형자산(특허, 소프트웨어, 개발비, ...)
  - 사용권자산 + 감가상각누계액
  - 부채(차입금, 매입채무, 미지급금, 리스부채, ...)
  - 자본(자본금, 이익잉여금, ...)
```

### 2.3 검증 체계
| 검증 | 조건 | 설명 |
|------|------|------|
| 열 검증 (R1) | Σ(R12~R90, 비현금) - R9 = 0 | 각 BS계정의 증감이 CF에 완전히 배분됨 |
| 행 검증 (A열) | Σ(F~끝) - E = 0 | 각 CF항목의 금액이 BS계정별 배분과 일치 |
| 현금 검증 | R87(순증감) = R89 - R88 | CF 합계가 현금 변동과 일치 |
| BS 검증 | R8 = R7 + R9 | 당기말 = 당기초 + 증감 |

### 2.4 연결 CF정산표 구조
```
컬럼: 과목 | 지배회사 | 종속1 | 종속2 | ... | 단순합산 | 연결조정 | 연결CF
```
- 각 회사별 별도 CF정산표 → 단순합산 → 연결조정(내부거래 제거 등) → 연결 CF

## 5. 화면 구성

### 5.1 화면 흐름
```
[시작/프로젝트] → [TB 업로드] → [CoA 매핑] → [CF정산표 편집] → [검증/완료] → [Excel 다운로드]
      ↑                                              ↓
      └──────── [프로젝트 저장/불러오기] ←────────────┘
```

### 5.2 주요 화면

#### 화면 1: 프로젝트 홈
- 새 프로젝트 생성 / 기존 프로젝트 불러오기
- 최근 프로젝트 목록

#### 화면 2: TB 업로드 & 파싱
- Excel/CSV 파일 드래그 앤 드롭
- 파싱 결과 미리보기 (계정코드, 계정명, 당기초, 당기말)
- 컬럼 매핑 (어떤 열이 계정코드/명/당기초/당기말인지 지정)

#### 화면 3: CoA 매핑
- 원장 계정 → BS 분류 매핑 테이블
- BS 분류: 자산(유동/비유동), 부채(유동/비유동), 자본
- CF 카테고리 기본 매핑 (자동 제안 + 수동 수정)
- 매핑 템플릿 저장/불러오기

#### 화면 4: CF정산표 메인 (핵심 화면)
- **상단 고정**: BS계정 헤더 (가로 스크롤, 당기초/당기말/증감 표시)
- **좌측 고정**: CF 항목 트리 (영업/투자/재무, 접기/펼치기)
- **본문**: 금액 입력 그리드 (행=CF항목, 열=BS계정)
- **검증 패널**:
  - 상단 R1행: 각 열의 검증값 (0이면 녹색, ≠0이면 빨간색)
  - 좌측 A열: 각 행의 검증값
  - 현금 검증 뱃지
- **도구 모음**:
  - 행 추가/삭제 (CF항목 커스텀)
  - 비현금거래 섹션 토글
  - 검증 요약 대시보드

#### 화면 5: CF 미리보기
- 완성된 현금흐름표 형태로 미리보기
- K-IFRS 양식

#### 화면 6: Excel 다운로드
- CF정산표 시트 + CF초안 시트 포함
- 서식 (색상, 테두리, 숫자 형식) 적용

### 6.2 계정 (TB에서 파싱)
```typescript
interface Account {
  id: string;
  code: string;                    // 계정코드
  name: string;                    // 계정명 (예: "건물")
  bsCategory: BSCategory;         // BS 분류
  openingBalance: number;          // 당기초 잔액
  closingBalance: number;          // 당기말 잔액
  change: number;                  // 증감 (자동 계산)
  debitCredit: 'DR' | 'CR';       // 차변/대변 구분
  columnIndex: number;             // 정산표 열 위치
}

type BSCategory =
  | '유동자산' | '비유동자산'
  | '유동부채' | '비유동부채'
  | '자본';
```

### 6.3 CF 항목 템플릿
```typescript
interface CFTemplate {
  sections: CFSection[];
}

interface CFSection {
  id: string;
  type: 'operating' | 'investing' | 'financing' | 'noncash';
  title: string;                   // "I. 영업활동으로 인한 현금흐름"
  items: CFItem[];
}

interface CFItem {
  id: string;
  label: string;                   // "감가상각비"
  level: number;                   // 들여쓰기 레벨 (0=대분류, 1=중, 2=소)
  isSubtotal: boolean;             // 소계 여부
  formula?: string;                // 소계 수식 (자식 합계)
  defaultMapping?: {               // 기본 BS계정 매핑
    accountPattern: string;        // 매핑 패턴 (예: "감가상각누계액*")
    sign: 1 | -1;                  // 부호
  };
  children?: CFItem[];
}
```

## 7. 핵심 알고리즘

### 7.1 검증 로직
```
// 열 검증 (각 BS계정별)
columnCheck(accountId) =
  Σ(entries where accountId) + Σ(nonCashEntries where accountId) - account.change
  → 결과가 0이면 PASS

// 행 검증 (각 CF항목별)
rowCheck(cfItemId) =
  Σ(entries where cfItemId) - cfItem.displayAmount
  → 결과가 0이면 PASS (소계 항목은 자식 합계 자동 계산)

// 현금 검증
cashCheck =
  CF_I + CF_II + CF_III + 환율효과 - (기말현금 - 기초현금)
  → 결과가 0이면 PASS
```

### 7.2 자동 매핑 제안
```
TB 계정명에서 키워드 매칭:
  "감가상각" → 조정: 감가상각비
  "대손" → 조정: 대손상각비
  "매출채권" → 영업활동 자산변동
  "토지", "건물", "기계" → 투자활동: 유형자산
  "차입금" → 재무활동: 차입금
  "리스부채" → 재무활동: 리스부채 상환
  ...
```
